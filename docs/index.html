Five lines to valid JSON

Pass a schema, get conformant output. Every time.

from llmjson import generate

schema = {"type": "object", "properties": {
    "name": {"type": "string"},
    "age":  {"type": "integer"}
}}

result = generate(
    model_id="mlx-community/Qwen2.5-14B-Instruct-4bit",
    prompt="Profile for a software engineer named Alice.",
    schema=schema,
)

print(result.text)    # {"name": "Alice", "age": 29}
print(result.is_valid) # True — always

How it works

1

Scan the vocabulary

VocabScanner classifies every token in the model's vocabulary into structural JSON types (braces, brackets, strings, numbers, literals). Cached to disk per tokenizer.

2

Track JSON context

A stack-based state machine follows every character produced. At each step it knows which JSON token types are valid next, based on the schema and current position.

3

Mask invalid tokens

The AND Gate sets logits of all structurally invalid tokens to -infinity before the model picks the next token. The model can only choose from tokens that keep the JSON valid.

4

Valid JSON, guaranteed

Since every generated token is constrained to be structurally valid, the output is always valid JSON conforming to your schema. No retries, no regex, no hope.

llmjson is a Python library for Apple Silicon (MLX). No server process, no API keys. Outlines and Guidance target CUDA/CPU. Ollama works but runs 10-25x slower on the same hardware. Pick the right tool for your setup.

CLI included

Pipe schemas, get JSON. Works in scripts and pipelines.

# Generate from a schema file
$ llmjson generate \
    --model mlx-community/Qwen2.5-14B-Instruct-4bit \
    --schema person.schema.json \
    --prompt "Profile for Alice, a senior engineer"

{"name": "Alice Chen", "age": 32, "email": "alice@example.com", "active": true}

# Verbose mode shows generation stats
$ llmjson generate --model ... --schema ... --prompt ... --verbose
# Steps: 47 | Time: 1.2s | Valid: True
